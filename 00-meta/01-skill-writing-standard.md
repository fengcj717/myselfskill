# Skill 写作规范

## 目标

统一 `myselfskill` 中所有 Skill 的写法，避免每个 Skill 都是散乱 Prompt。

一个合格 Skill 应该做到：

1. 触发条件明确。
2. 输入信息清楚。
3. 执行步骤稳定。
4. 输出格式可复用。
5. 安全边界清楚。
6. 能被 ChatGPT、Codex、Claude Code 等工具理解和执行。

## 推荐目录结构

每个 Skill 建议使用独立目录：

```text
skill-name/
  SKILL.md
  README.md              # 可选：给人看的说明
  examples/              # 可选：示例输入输出
  templates/             # 可选：模板文件
  scripts/               # 可选：脚本，必须有安全说明
```

入口型、规范型、索引型 Skill 放在：

```text
00-meta/
```

业务型 Skill 按主题放置，例如：

```text
content-creation/
enterprise-ai-training/
knowledge-base-management/
github-obsidian-workflow/
startup-analysis/
```

## SKILL.md 标准字段

每个 `SKILL.md` 建议包含以下字段。

### 1. Skill 名称

用一句话说明这个 Skill 解决什么问题。

示例：

```md
# 小红书企业 AI 培训内容生产 Skill
```

### 2. 目标

说明这个 Skill 的最终产出。

不要只写“帮助用户写小红书”，要写清楚输出对象和使用场景。

示例：

```md
## 目标

帮助 Jeffery 围绕企业 AI 培训方向，持续生成适合小红书发布的选题、图文结构、短视频脚本、封面文案和发布前质检清单。
```

### 3. 触发条件

说明什么情况下应该调用这个 Skill。

示例：

```md
## 触发条件

当用户提到以下关键词时调用：

- 小红书
- 企业 AI 培训
- AI 培训课
- 主页定位
- 选题
- 短视频脚本
- 图文笔记
```

### 4. 输入要求

说明执行前需要哪些信息。

示例：

```md
## 输入要求

优先读取：

- 目标受众
- 内容平台
- 内容主题
- 是否用于获客
- 是否需要承接商单或培训交付
```

如果信息不全，能推断就推断，不要频繁追问。

### 5. 执行流程

这是 Skill 的核心。

要求步骤明确，避免泛泛而谈。

示例：

```md
## 执行流程

1. 判断内容目标：涨粉、获客、建立专业感、承接咨询或培训。
2. 判断目标人群：老板、管理者、HR、业务负责人、普通员工。
3. 生成 5-10 个选题。
4. 每个选题给出痛点、标题、内容结构、转化钩子。
5. 输出发布前质检清单。
```

### 6. 输出格式

固定输出结构，方便复用。

示例：

```md
## 输出格式

- 选题名称
- 目标人群
- 核心痛点
- 标题建议
- 图文结构
- 视频脚本
- 评论区引导
- 私信承接话术
- 发布前质检
```

### 7. 安全边界

凡是涉及外部代码、插件、浏览器、账号、API Key、文件写入、爬虫、平台抓取，都必须写安全边界。

示例：

```md
## 安全边界

不得绕过登录、验证码、反爬或平台权限限制。
不得要求用户粘贴账号密码、cookie、token。
涉及 GitHub 写入前，优先创建分支或 PR，不直接覆盖主分支。
```

### 8. 反向沉淀规则

说明执行完后如何补充 Skill。

示例：

```md
## 反向沉淀

如果本次任务产生新的稳定流程、模板、质检项或案例，应建议更新本 Skill。
```

## Skill 质量检查清单

创建或更新 Skill 后，检查：

- 是否有明确触发条件？
- 是否能减少下次沟通成本？
- 是否包含具体执行步骤？
- 是否有稳定输出格式？
- 是否避免空泛原则？
- 是否符合 Jeffery 的长期方向？
- 是否有安全边界？
- 是否适合沉淀到 `myselfskill`，而不是只放在普通知识库？

## Skill 与普通笔记的区别

普通笔记记录知识。

Skill 记录可执行流程。

如果内容只是观点、素材、案例、文章摘录，放入 Obsidian 知识库。

如果内容能指导 ChatGPT / Codex / Claude Code 重复完成某类任务，放入 `myselfskill`。

## 命名规范

文件夹名使用英文小写和连字符：

```text
skill-router
content-quality-check
xiaohongshu-enterprise-ai
obsidian-github-writeback
```

文件标题可以使用中文，便于阅读。

## 版本维护

每次更新 Skill 时，建议在文件底部添加：

```md
## 更新记录

- 2026-06-23：创建初版。
```

不要为了形式频繁改版本号。只有当触发条件、流程、输出格式发生明显变化时才记录。
