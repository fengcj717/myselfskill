# 消息处理与注意力防火墙（Message Triage）

## 目标

这个 Skill 用于处理外部信息流，不做简单摘要，而是判断一条消息是否值得进入用户的注意力系统。

核心目标：

> 让外部信息经过筛选之后，只留下三类资产：行动、判断、素材。

适用来源：

- Telegram 群消息
- 微信群消息
- 即刻动态
- 公众号文章
- 小红书内容
- 朋友转发
- 招聘信息
- AI 工具动态
- 播客摘录
- 用户手动粘贴的任意消息

## 用户当前主线

默认以 Jeffery 当前主线作为判断标准：

1. 企业 AI 培训 / 企业 AI 咨询
2. AI 工作流 / Agent / RAG / 知识库
3. 小红书 / 公众号 / 博客内容获客
4. 真实企业痛点、岗位痛点、用户原话
5. 个人知识库 / Obsidian / GitHub / Codex 工作流
6. 数据开发、数据分析、企业 AI 转型能力

一条消息如果和这些主线没有关系，即使有趣，也默认降权。

## 五类消息分类

### 1. 垃圾消息

定义：只制造刺激，没有事实、结构、行动价值。

特征：

- 标题党
- 情绪煽动
- 八卦争议
- 无上下文的爆料
- “赶紧看”“太炸裂”“普通人再不上车就晚了”
- 看完没有任何可执行信息

处理：删除、点不感兴趣、取关、屏蔽来源。

### 2. 焦虑消息

定义：不是完全没信息，但主要作用是让用户不安。

特征：

- 强调别人已经成功
- 强调窗口期快没了
- 展示收入、流量、结果，但没有过程细节
- 让人觉得落后，但不给下一步动作
- 把复杂事情包装成轻松复制

处理：不收藏全文，只提炼真实市场信号；必要时标记为焦虑源。

### 3. 观察消息

定义：暂时不需要行动，但提供了值得观察的信号。

特征：

- 用户需求反复出现
- 某个岗位反复讨论同一个问题
- 某类工具被大量使用或吐槽
- 某个平台内容风向变化
- 某个企业场景反复出现

处理：进入素材库、痛点库、趋势观察库。

### 4. 关键消息

定义：能改变判断，能形成内容、案例、服务模块、SOP 或培训素材。

至少满足两个条件：

- 和当前主线强相关
- 能改变判断
- 能形成内容选题
- 能变成服务模块
- 能补充真实案例
- 能帮助设计培训、SOP、产品包
- 能成为未来反复使用的认知模型

处理：进入知识库，并改写为结构化笔记。

### 5. 行动消息

定义：不是用来收藏的，而是需要执行。

特征：

- 工具需要测试
- 客户问题需要跟进
- 选题可以立即写
- 案例可以拆成内容
- 有企业 AI 培训线索
- 有明确能力补课方向

处理：进入任务系统，而不是知识库。

## 六问过滤器

处理任何消息前，按顺序判断：

1. 它和当前主线相关吗？不相关，默认丢弃。
2. 它提供事实、案例、原话，还是只提供情绪？只有情绪，降权。
3. 它让我更清晰，还是更焦虑？只制造焦虑，不入库。
4. 它能不能变成内容、案例、服务、SOP、培训素材？能，保留。
5. 它有没有明确下一步动作？有，进任务系统，不要只收藏。
6. 三个月后的我还会需要它吗？不会，别存。

## 处理原则

- 不把“有趣”当成“有用”
- 不把“焦虑”当成“机会”
- 不把“收藏”当成“行动”
- 不被标题党带节奏
- 优先保留真实原话、具体场景、可验证信号
- 所有保留信息都要尽量转化成内容、案例、培训素材、咨询机会或行动任务

## 长期判断句

> 你不是在刷信息，你是在训练未来的自己。

> 信息输入结构，决定问题意识；问题意识，决定 AI 使用方式；AI 使用方式，决定最终沉淀的能力。
